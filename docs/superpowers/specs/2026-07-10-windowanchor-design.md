# WindowAnchor — Design

**Date:** 2026-07-10
**Goal:** A free, open-source macOS (Apple Silicon) window manager that replicates the
Windows 11 Snap Layouts experience — drag a window to the top-center of the screen and a
flyout of layout presets appears; drop the window into a cell to snap it there. Fully
customizable, with a modern Liquid Glass look.

## Users

People switching from Windows (or using both) who miss Windows 11 snap behavior on macOS.
Must be installable by a non-technical user from a DMG.

## Core behaviors

1. **Snap Layouts flyout (the headline feature).** While dragging a window, moving the
   cursor into a small hot zone at the top-center of the screen shows a flyout with layout
   presets (halves, 2/3+1/3, thirds, left half + stacked right, quarters, 25/50/25).
   The cell under the cursor highlights; releasing the mouse snaps the dragged window into
   that cell. Holding **Option** during a drag also summons the flyout at the cursor
   (configurable).
2. **Edge snapping.** Dragging to the left/right screen edge previews and snaps to that
   half; corners snap quarters; the top edge maximizes (all toggleable).
3. **Snap Assist.** After snapping into a layout cell, a panel appears in the next empty
   cell listing the user's other open windows (app icon + title). Clicking one snaps it
   into that cell and advances to the next empty cell. Esc/click-outside dismisses.
4. **Customization.** Settings window with: enable/disable + reorder layout presets,
   a visual custom-layout editor (BSP splits: split any cell horizontally/vertically,
   drag dividers to resize, delete cells), window gap + outer padding, hover delay,
   trigger toggles, launch at login.

## Architecture

Menu-bar-only app (`LSUIElement`), AppKit lifecycle with SwiftUI views hosted in panels.

- **DragMonitor** — listen-only `CGEventTap` on mouse down/drag/up. At mouse-down it
  captures the AX window under the cursor; a drag is confirmed when the window's AX
  position tracks the mouse delta. Emits `dragBegan/dragMoved/dragEnded` with the window
  and cursor location (CG top-left coordinates).
- **AXWindow** — wrapper over `AXUIElement`: read/write frame, title, raise, owner PID.
  Requires the Accessibility permission (`AXIsProcessTrustedWithOptions` prompt flow,
  polled until granted, with a menu-bar onboarding state).
- **SnapEngine** — pure geometry: converts a normalized layout cell + screen visible
  frame + gap settings into an absolute CG rect; applies frames via AXWindow.
- **EdgeZones** — hit-testing for edge/corner/top zones on the screen under the cursor.
- **LayoutPickerController** — non-activating `NSPanel` (glass) shown near the top-center
  hot zone. Because the user is mid-drag, hover highlighting is driven by the event tap's
  cursor position, not real mouse events. Reports the hit cell on mouse-up.
- **SnapPreviewController** — translucent rounded preview of the target frame (edge snap).
- **SnapAssistController** — post-snap panel; enumerates other windows via
  `CGWindowListCopyWindowInfo` (layer 0, on-screen), matched to AX windows by PID+title.
- **Model** — `SnapLayout` (id, name, cells: normalized rects, optional BSP tree for
  editing, enabled flag), built-in presets + user layouts; `Preferences`
  (ObservableObject persisted to UserDefaults, layouts as JSON).
- **StatusItemController** — menu bar icon/menu: enable toggle, settings, permission
  status, quit.
- **Settings UI** — SwiftUI window, tabs: General, Layouts (incl. editor), Snapping,
  About. Liquid Glass styling (`.glassEffect()` on macOS 26, material fallback earlier).

### Coordinates

Internal standard: CG global coordinates (top-left origin, y down) — what the event tap
and AX use. Converted to AppKit (bottom-left origin) only when positioning NSPanels.

## Non-goals (v1)

Keyboard-shortcut snapping, live window thumbnails in Snap Assist (needs Screen
Recording permission — icons/titles only), restoring pre-snap size on drag-away,
window animation, Intel builds.

## Distribution

SPM release build (arm64) → assembled `.app` bundle (Info.plist, .icns generated by
script) → ad-hoc codesign → DMG via `hdiutil` → GitHub Release. README documents the
Gatekeeper right-click-open / `xattr -cr` step since the app is not notarized (no paid
Developer ID), plus the Accessibility permission grant.

## Testing

Geometry (SnapEngine, BSP tree → rects, edge zone hit tests) covered by XCTest unit
tests run in CI-less local build. Interactive behavior verified manually by launching
the built app.
